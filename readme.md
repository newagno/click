# 🤖 Daily Claim Bot (2026 Edition)

Високонадійне, production-ready рішення для автоматичного щоденного збору поінтів **KEY** на сайті [Incrypted Account](https://incrypted.com/ua/account/) з обходом захисту Cloudflare (Turnstile) та інтеграцією сповіщень у Telegram.

---

## 🚀 Основні переваги та архітектурні рішення

1. **Клієнтський рівень (SeleniumBase у режимі UC)**:
   - Використовує **Undetected ChromeDriver (UC)** від SeleniumBase, що маскує внутрішні змінні WebDriver (`navigator.webdriver`, хроміум-специфічні об'єкти тощо).
   - Емуляція реального браузера (стелс-технологія) — замість стандартного безголового режиму (`headless=True`), який легко детектується Cloudflare, бот запускається у віртуальному фреймбуфері (`xvfb-run` на GitHub Actions), що виглядає для систем захисту як справжнє GUI-вікно.
2. **Динамічний жест Swipe (Drag-and-Drop)**:
   - Щоденна нагорода на сайті збирається перетягуванням повзунка (slider swipe). Бот автоматично зчитує ширину треку через JS-контекст та динамічно розраховує правильний офсет перетягування, імітуючи реальний рух миші користувача через `ActionChains`.
3. **Розумна обробка статусів та запобігання спаму**:
   - Бот автоматично розпізнає стан нагороди: якщо вона вже зібрана, він парсить точний таймер зворотного відліку (наприклад, `09:29:27`) та переходить у режим очікування.
   - Опція `SILENT_ON_COOLDOWN` у файлі `.env` дозволяє вимкнути спам у Telegram кожні 3 години, якщо нагорода вже була забрана, залишаючи при цьому детальні логи у GitHub Actions.
4. **Кросплатформність**:
   - Автоматично виявляє наявність Brave Browser на локальній Windows-системі. На Linux/GitHub Actions бот автоматично перемикається на стандартний Google Chrome, не потребуючи додаткових конфігурацій.

## 🛠️ Швидкий старт (Локальний запуск)

### 1. Підготовка оточення
Переконайтеся, що на вашому комп'ютері встановлено **Python 3.10+** та веб-браузер Google Chrome / Brave.

```bash
# Клонуйте репозиторій та перейдіть у робочу директорію
cd click

# Створіть та активуйте віртуальне оточення Python
python -m venv venv
# Для Windows:
venv\Scripts\activate
# Для Linux/macOS:
source venv/bin/activate

# Встановіть необхідні залежності
pip install -r requirements.txt
```

### 2. Налаштування `.env`
Скопіюйте шаблон налаштувань та заповніть ваші дані:
```bash
cp .env.example .env
```

Відкрийте створений файл `.env` і впишіть:
* `INCRYPTED_EMAIL` та `INCRYPTED_PASSWORD` — ваші логін/пароль від Incrypted.
* `TG_BOT_TOKEN` та `TG_CHAT_ID` — дані вашого Telegram-бота для сповіщень.
* `RESIDENTIAL_PROXY` *(рекомендовано)* — резидентський проксі у форматі `user:pass@ip:port`.
* `SILENT_ON_COOLDOWN=True` — тихий режим, щоб не спамити повідомленнями "Ще не час".

### 3. Запуск логування та перевірка
Запустіть скрипт:
```bash
python src/main.py
```

## ☁️ Налаштування та деплой у GitHub Actions (Безкоштовно, 24/7)

Бот повністю налаштований для щоденного безкоштовного запуску за розкладом (cron) через GitHub Actions. Робочий процес описано у файлі `.github/workflows/daily-claim.yml`.

### Як додати секрети у ваш GitHub-репозиторій:
1. Перейдіть до вашого репозиторію на GitHub.
2. Відкрийте вкладку **Settings** -> **Secrets and variables** -> **Actions**.
3. Натисніть кнопку **New repository secret** та додайте наступні змінні:

| Назва секрету | Значення |
| :--- | :--- |
| `INCRYPTED_EMAIL` | Електронна пошта від вашого акаунту |
| `INCRYPTED_PASSWORD`| Пароль від вашого акаунту |
| `TG_BOT_TOKEN` | Токен Telegram-бота (від `@BotFather`) |
| `TG_CHAT_ID` | Ваш особистий Telegram Chat ID (від `@userinfobot`) |
| `RESIDENTIAL_PROXY` | *(Необов'язково)* Резидентські проксі `user:pass@ip:port` |

*Робочий процес запускається автоматично за кроном кожні 3 години (щоб гарантовано не пропустити claim), а також ви можете запустити його вручну через вкладку **Actions -> Daily Incrypted Claim -> Run workflow**.*

## 🛡️ Секрети обходу Cloudflare та безпека акаунту (Аналітика 2026)

### 1. Як найкраще обходити Cloudflare Turnstile у 2026 році?
* **Undetected ChromeDriver (UC)**: Базові бібліотеки на кшталт стандартного Selenium або Puppeteer без stealth-плагінів провалюють Turnstile IUAM виклик миттєво через розкриття прапорців автоматизації (`cdc_adoQpoasnfa76pfcZLmcfl_Array` тощо). SeleniumBase UC динамічно патчить ChromeDriver в оперативній пам'яті.
* **Гібридний Headless-режим**: Запуск браузера за допомогою `xvfb-run` (Virtual Framebuffer) на GitHub Actions — найкраща практика. Cloudflare перевіряє наявність графічного рендерингу (WebGL, Canvas, наявність монітора). Віртуальний дисплей повністю задовольняє ці перевірки, зберігаючи стелс-статус.

### 2. Чи варто використовувати Residential Proxies або Cloudflare Workers?
* **Резидентські проксі (Residential Proxies)**: **ОБОВ'ЯЗКОВО для стабільності**. IP-адреси хостинг-провайдерів та хмарних серверів (таких як GitHub Enterprise, AWS, DigitalOcean) мають максимальний рівень підозрілості (Bad IP Score) у базах Cloudflare (IP Intelligence). Без проксі ви будете постійно отримувати нескінченні капчі або 403 помилки. Резидентські проксі мають реальні домашні IP (ISP) і проходять перевірку безперешкодно.
* **Cloudflare Workers**: Не підходять для автоматизації фронтенду браузером, оскільки вони працюють на рівні V8 Isolate та призначені для HTTP-маршрутизації та API, а не для повноцінного клієнтського рендерингу важких JS-капч та драг-енд-дроп логіки.

### 3. Як мінімізувати ризики бана акаунту?
* **Рандомізація часу (Jitter)**: У робочому процесі GitHub Actions запуски вже налаштовані на випадкові хвилини години (`15 */3 * * *`). Для максимальної безпеки додано exponential backoff з випадковою затримкою між спробами повтору.
* **Статичний Fingerprint**: Використання якісних проксі зі стабільною геолокацією запобігає тригерам системи безпеки про "підозрілу зміну локації" (наприклад, коли ви заходите з Києва локально, а бот через хвилину з датацентру GitHub у США).

## 📊 Порівняння альтернативних технологій

| Технологія | Обхід Cloudflare (2026) | Складність підтримки | Споживання ресурсів | Висновок |
| :--- | :--- | :--- | :--- | :--- |
| **SeleniumBase (UC) (ОБРАНО)** | **Високий (5/5)** | **Низька** | Середнє | **Найкращий баланс стабільності, нативності та простоти автоматизації.** |
| Standard Puppeteer / Playwright | Низький (1/5) | Висока (потрібен stealth) | Низьке | Постійні блокування на етапі ініціалізації Turnstile. |
| Pyppeteer + stealth | Середній (3/5) | Висока (часті баги) | Низьке | Застаріла кодова база, не підтримує нові алгоритми Turnstile 2026 року. |
| Selenium + UC (Raw) | Високий (4/5) | Середня | Середнє | Добре, але вимагає багато шаблонного коду для обробки фреймів та драг-енд-дроп подій. |
